# Aula 01

Engenharia de software → Se trata de aplicar métodos consolidados e boas práticas para **desenvolver, operar e manter sistemas de software com alta qualidade, eficiência e confiabilidade**

o software começa no problema →  com esse problema temos que entender o problema e como resolver o problema, faremos o diagrama desse problema → quando tiver diagramado e entendível → e colocar a mão na massa → Produto (final)

→ Podemos dividir a construção do software em partes (metodologias) ou só contruir e fds normalmente sozinho

→ A disciplina vai se basear nas fases para chegar no Produto final.

Manunteção de software → Arrumar erros/bugs ou fazer melhorias que podem ser feitas nesse codigo

Fases do caminho da realização do software

1º → Requisitos (expressar os problemas que o usuário precisa resolver, aquilo que vamos desenvolver, quanto melhor especificado os requisitos melhor vc podera estimar o tempo para ser feito)

→ existe dois tipos de requisitos, funcionais (assistir video) e não funcionais (assistir em alta resolução), ou seja o não funcional complementa os funcionais.

 2º → Análise → idealização do software, fazer diagramas que representa os requisitos etc

3º → Projeto → Deixa de ser idealização e passa a ser concreto, tem q estar definido o que vai usar, qual tecnologia qual versão etc

4º → Implementação → Codificar os requisitos baseado nas analises e no projeto

5º → Teste

6º → Entrega o software

7º → Manutenção do software

Quando falamos do software estamos falando do produto.

Quando falamos das fases da realização do software estamos falando de processos.
